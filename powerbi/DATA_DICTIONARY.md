# Power BI Data Dictionary

All customer-level decision outputs in this folder are generated by `python run_pipeline.py`.
The source is the public IBM/Kaggle Telco Customer Churn snapshot. It has no event timestamp,
so these files represent a reproducible portfolio case study rather than a live production feed.

## customer_retention_queue.csv

- Grain: one held-out customer used for out-of-sample evaluation.
- Refresh source: selected pipeline scores created by `src/churn_pipeline.py`.
- Expected use: Retention Action Queue and Customer Risk Portfolio pages.

Key columns:

| Column | Definition |
|---|---|
| `customerID` | Traceability key; never used as a model feature. |
| `churn_probability` | Selected model's probability score. It is not presented as calibrated. |
| `predicted_churn_flag` | `1` when probability is at or above the selected intervention threshold. |
| `risk_segment` | Business-linked Low, Medium, or High Risk segment. |
| `retention_priority_score` | Transparent score: 60% churn probability, 25% value proxy, 15% urgency. |
| `priority_level` | Priority 1, Priority 2, Priority 3, or Monitor. |
| `customer_value_proxy` | Relative proxy from MonthlyCharges and TotalCharges scaled to training-data 95th percentiles; not CLV. |
| `intervention_urgency` | Relative proxy from shorter tenure and month-to-month contract. |
| `primary_reason` | Highest-ranked observed risk indicator present for the customer. |
| `secondary_reason` | Second observed risk indicator, when available. |
| `suggested_action` | Rule-based intervention hypothesis, not a proven treatment. |
| `suggested_success_metric` | Measurement proposed for a future intervention test. |

## risk_segment_summary.csv

- Grain: one row per risk segment.
- Refresh source: held-out customer retention queue.
- Expected use: Executive Retention Overview and Customer Risk Portfolio.
- Measures: customer count/share, average probability, observed churn rate, average tenure,
  average monthly charges, and contract mix.

## model_performance_summary.csv

- Grain: one row per evaluated model.
- Refresh source: held-out test evaluation after validation-only threshold selection.
- Expected use: model evidence and decision-error tooltips.
- Important fields: selected model, decision threshold, churn precision/recall/F1,
  PR-AUC, ROC-AUC, and confusion counts.

## churn_driver_summary.csv

- Grain: one observed indicator for one original model feature.
- Refresh source: permutation importance plus historical churn-rate association.
- Expected use: Churn Drivers page.
- Caveat: associations are descriptive and must not be interpreted as causal effects.

## summary_by_contract.csv

- Grain: one contract type.
- Refresh source: complete cleaned historical dataset.
- Expected use: contract portfolio context.

## summary_by_payment_method.csv

- Grain: one payment method.
- Refresh source: complete cleaned historical dataset.
- Expected use: payment portfolio context.

## summary_by_internet_service.csv

- Grain: one internet-service type.
- Refresh source: complete cleaned historical dataset.
- Expected use: service portfolio context.

## Scope warning

The three summary tables cover the full historical dataset, while customer-level scoring and
risk summaries cover the held-out test population. Do not combine their customer counts into
one KPI without an explicit population filter.
